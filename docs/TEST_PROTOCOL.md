# UNS 実車テスト・プロトコル v1.0

## 目的
「感覚」を定量データに変換し、修正優先順位を客観的に決定する。
テスターは開発者・PMの両者が実施可能。専門知識不要。

---

## テスト環境

| 路線 | 区間 | 理由 |
|---|---|---|
| 山手線 | 渋谷 → 新宿（内回り） | 最大乗降数・混雑の標準環境 |
| 大江戸線 | 新宿 → 六本木 | 高周波ブレーキ音が最も顕著な路線 |
| 銀座線 | 渋谷 → 表参道 | 狭小トンネルによる低周波共鳴が強い |

**実施時間帯（優先）**
- 朝: 07:30–09:00（ピーク・最大負荷）
- 夜: 20:00–22:00（ジム帰り想定・疲労蓄積状態）

**使用機材**
- iPhone 14 以降（テスター A）
- Android Pixel 7 以降（テスター B）
- Apple Watch Series 7 以降（あれば）
- ノイズキャンセリングイヤホン（AirPods Pro または Sony WH-1000XM5）

## テスト項目一覧

### カテゴリ 1: 音の馴染み（Noise Ambient 品質）

| ID | テスト内容 | 評価軸 | 合格基準 |
|---|---|---|---|
| A-01 | 乗車直後、外部騒音がドローン音に「飲み込まれる」感覚があるか | 主観 1–5点 | 平均 3.5以上 |
| A-02 | ドローン音が「突然始まった」という違和感があるか | 二択（あり/なし） | 「あり」が20%未満 |
| A-03 | 停車時（静かな瞬間）に音が不自然に大きく感じないか | 二択 | 「違和感あり」が10%未満 |
| A-04 | 3分経過後、外部音と内部音の境界を意識しているか | 二択 | 「意識している」が30%未満 |
| A-05 | 10分経過後、「静かになった」という感覚があるか | 二択（Y/N） | 「Yes」が70%以上 |

**記録方法:** テスト終了直後にNotesアプリに 1〜5 の数字と感想を入力。

---

### カテゴリ 2: 高音スパイク検知（ブレーキ音マスキング）

| ID | テスト内容 | 評価軸 | 合格基準 |
|---|---|---|---|
| B-01 | ブレーキ音に対して風鈴音が重なったか | 二択 | 観測セッション中3回以上 |
| B-02 | 風鈴音のタイミングは「ブレーキ音と同時」か「遅かった」か | 三択（同時/遅れ/不明） | 「遅れ」が20%未満 |
| B-03 | 風鈴音は不自然に大きくないか（耳障りでないか） | 二択 | 「大きすぎ」が0% |
| B-04 | ブレーキ音発生時にShieldがフラッシュしたか（画面確認） | 二択 | 「した」が80%以上 |
| B-05 | ブレーキ音の「嫌な感じ」が軽減されたか | 主観 1–5点 | 平均 3.0以上 |

**大江戸線特記:** 新宿西口〜都庁前〜六本木の区間でB-01〜B-05を集中実施。

### カテゴリ 3: 歩行〜乗車遷移の違和感

| ID | テスト内容 | 評価軸 | 合格基準 |
|---|---|---|---|
| C-01 | 改札通過後にMind Weather通知が来たか（テスト日のみ確認） | 二択 | 当日送信確認 |
| C-02 | 「Sanctuaryを展開する」タップから音が出るまでの体感秒数 | 秒数記録 | 2秒以内 |
| C-03 | 歩行中（ホーム移動）に音が途切れたか | 二択 | 「途切れた」が0% |
| C-04 | 電車乗車後、音の質（ドローン音量）が変化したか | 三択（増えた/変わらず/減った） | 「増えた」が60%以上 |
| C-05 | 下車後、「解除する」を押さずに自然に使用終了した際の違和感 | 二択 | 記録のみ（改善要件化） |

### カテゴリ 4: バックグラウンド維持（OS強制終了チェック）

| ID | テスト内容 | 手順 | 合格基準 |
|---|---|---|---|
| D-01 | セッション中にSNSアプリ（Instagram）を5分使用 | 手動実施 | 音が継続している |
| D-02 | セッション中に電話着信をシミュレート（もしくは着信） | 手動実施 | 着信後に音が再開する |
| D-03 | ロック画面でNow Playingバッジが表示されているか | 手動確認 | 「Sanctuary 稼働中」が表示 |
| D-04 | D-03のNow Playing情報が動的更新されているか | 30秒後に再確認 | 経過時間または強度が変化 |
| D-05 | セッション中にアプリを強制終了（スワイプアウト）した際の動作 | 手動実施 | 音が停止し通知なし（記録のみ） |

## 優先度マトリクス（修正判断基準）

| 重大度 | 条件 | アクション |
|---|---|---|
| P0 (即日修正) | A-02「違和感あり」30%超、C-03「途切れた」発生 | リリースブロック |

## Debug Panel 活用手順（実車中の調整）

```
1. SanctuaryScreen 下部「▼ DEBUG」をタップ
2. セッション開始後、以下の順で調整:
   a. droneVolMultiplier: 0.8 → 1.2 → 1.6 と段階的に上げ、
      「音が馴染む」ポイントを記録
   b. lowFreqRampMs: 500ms → 300ms → 800ms を比較
      「急に始まった感じ」の有無を記録
   c. highFreqThreshold: 0.72 (default) → 0.60 → 0.85 で
      ブレーキ音への反応感度を比較
3. 「CURRENT STATE」セクションの値を記録シートに転記
4. RESET TO DEFAULT で次のテスターへ引き継ぐ
```

## テスト後アクション

1. 記録シートを `docs/test-results/YYYY-MM-DD_路線名.md` で保存
2. P0/P1 項目は当日中に GitHub Issue 化
3. Debug Panel のベスト設定値は `constants/theme.ts` の `NOISE_THRESHOLD` に反映
4. 3セッション分のデータが揃ったら週次レビューで優先度を確定
